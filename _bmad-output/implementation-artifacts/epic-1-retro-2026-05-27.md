## 4. Retrospective Discussion Summary

### What Went Well
- **Sleek HUD Layout & Visual Polish:** The Outfit and JetBrains Mono typography, custom glassmorphism, responsive navigation dock, and hardware-accelerated CRT scan-sweep transitions exceeded UX expectations and set a high standard for quality.
- **Robust Database Setup & Migration Flow:** Scaffolding Postgres with `pgvector` and creating a custom database migration runner ensures all database schemas are cleanly versioned and automatically updated at application startup.
- **Preventative Testing:** Colocating tests with components and routes immediately surface regression errors.

### Challenges & Struggles
- **Database Lifespan Lifecycle and Error Handling:** The initialization of `psycopg_pool` inside the lifespan context had edge cases where pool failures raised `PoolClosed` instead of `OperationalError`, causing health endpoints to bypass None checks. These were resolved by cleaning up initialization error cascades.
- **Security Vulnerabilities Identified in Review:** The initial CORS setup allowed wildcard headers (`allow_headers=["*"]`), and health check routers leaked raw exception traces. We corrected these by restricting CORS to `http://localhost:5173` and implementing structured error envelopes.
- **Autosave Race Conditions & Browser Cancellation:** Story 1.3 had race conditions where concurrent in-flight profile saves could override each other or fail to flush properly on page unmount. These were fixed by using React refs to capture the latest values and executing synchronous fetch calls on unmount.
- **Locale-Safe Shortcuts:** Keyboard shortcuts based on `e.key` were found to be layout-dependent. They were updated to layout-independent codes using `e.code.match(/^Digit([1-5])$/)`.
